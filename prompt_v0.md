# Структурированный промпт V0

## Назначение промпта

Промпт используется как подготовительный этап SPDD. Его цель - проанализировать курсовой проект TextGuard AI, выделить границы backend-разработки и не смешивать мою часть работы с frontend, обучением моделей и подготовкой датасетов.

## REASONS Canvas

### Requirements

- Изучить структуру проекта TextGuard AI.
- Определить, какие части относятся к backend, frontend, ML-моделям и данным.
- Зафиксировать мою область: серверная B2B-обвязка вокруг существующих `ClassifierService` и `DetoxService`.
- Выявить endpoint-ы, которые нужны клиентскому приложению и внешним B2B-интеграциям.
- Сформировать список backend-файлов, которые могут быть созданы или изменены.

### Entities

- `TextGuard AI` - сервис AI-модерации и детоксикации текста.
- `backend` - моя зона разработки.
- `FastAPI app` - приложение, регистрирующее роутеры и CORS.
- `ClassifierService` - готовый сервис классификации токсичности.
- `DetoxService` - готовый сервис переписывания токсичного текста.
- `API key` - ключ клиента для доступа к B2B API.
- `Workspace` - демонстрационная сущность клиента.
- `SPDD prompt` - версионируемый артефакт разработки.

### Approach

- Сначала провести анализ проекта, а не сразу генерировать код.
- Не изменять логику обучения моделей и локальные веса.
- Строить backend как тонкую продуктовую обвязку над существующими ML-сервисами.
- Сохранять совместимость со старым endpoint `/api/moderate`.

### Structure

- Проанализировать `README.md`, `backend/main.py`, `backend/services/`, `backend/routes/`.
- Зафиксировать будущие или уточняемые файлы:
  - `backend/deps.py`;
  - `backend/store.py`;
  - `backend/plans.py`;
  - `backend/user_store.py`;
  - `backend/routes/v1_moderation.py`;
  - `backend/routes/api_keys.py`;
  - `backend/routes/subscriptions.py`;
  - `backend/routes/dashboard.py`;
  - `backend/routes/auth.py`;
  - `backend/routes/keys.py`.

### Operations

- Найти текущие API-контракты.
- Описать роли Bearer API key, X-API-Key и X-Admin-Token.
- Сформировать будущие маршруты для анализа текста, ключей, тарифов и dashboard.
- Зафиксировать ограничения, которые нельзя нарушать.

### Norms

- Не приписывать backend-задаче разработку frontend.
- Не переписывать ML-сервисы без необходимости.
- Использовать русскоязычные сообщения ошибок.
- Делать код простым для демонстрационного курсового проекта.

### Safeguards

- Если модель уже загружается локально, не добавлять внешние облачные вызовы.
- Если endpoint уже используется frontend, не ломать формат ответа.
- Если ключ невалиден или лимит исчерпан, не запускать инференс.

## Текст промпта

Проанализируй проект TextGuard AI и подготовь план разработки только backend-части. Frontend, обучение моделей, подготовка датасетов и локальные веса не разрабатываются мной. Нужно определить, какие backend-файлы отвечают за FastAPI-приложение, авторизацию, API-ключи, тарифы, dashboard и вызов `ClassifierService`/`DetoxService`, а также зафиксировать ограничения перед генерацией кода.
