# Структурированный промпт V6

## Назначение промпта

Промпт используется для генерации тестовых сценариев backend без обязательной реализации автотестов.

## REASONS Canvas

### Requirements

- Сформировать функциональные сценарии проверки backend.
- Покрыть успешный анализ текста.
- Покрыть токсичный и безопасный текст.
- Покрыть ошибки авторизации.
- Покрыть лимиты тарифов.
- Покрыть управление ключами, тарифами и dashboard.
- Покрыть legacy endpoint и admin endpoint.

### Entities

- `B2B analyze request`
- `Legacy moderate request`
- `API key lifecycle`
- `Subscription flow`
- `Dashboard flow`
- `Auth flow`
- `Admin flow`
- `Error response`

### Approach

- Описать сценарии в Markdown.
- Для каждого сценария указать вход, шаги и ожидаемый результат.
- Не требовать запуска тяжелых ML-моделей в тестах документации.
- Отделить функциональную проверку API от качества ML-модели.

### Structure

- Создать `test_scenarios.md`.
- Разделить сценарии на группы:
  - модерация;
  - авторизация;
  - ключи;
  - тарифы;
  - dashboard;
  - legacy;
  - admin;
  - устойчивость.

### Operations

- Описать curl/HTTP-смысл сценариев.
- Указать ожидаемые HTTP-статусы.
- Указать ожидаемые поля JSON.
- Указать ограничения демонстрационной реализации.

### Norms

- Не писать фиктивные результаты метрик как реальные замеры.
- Не скрывать ограничение in-memory store.
- Не приписывать тестам полноценный production-аудит.

### Safeguards

- Если модель недоступна локально, проверять только API-контракт с mock/stub не выдавая это за ML-проверку.
- Если ключ отсутствует, сценарий должен завершаться до инференса.

## Текст промпта

Сформируй `test_scenarios.md` для backend TextGuard AI. Опиши функциональные сценарии проверки `/api/v1/moderation/analyze`, `/api/moderate`, `/api/api-keys`, `/api/subscriptions`, `/api/dashboard`, `/api/auth`, `/api/admin/keys`, ошибок 400/401/403/429, лимитов тарифа, журналирования запросов и корректности JSON-полей. Не утверждай, что это production-тестирование; это сценарии для лабораторной проверки.
